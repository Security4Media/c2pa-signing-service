Fix: Send Trait Issue in package_and_sign_processor¶

Problem¶

The async function sign_fragmented had a Send trait issue:

error: future cannot be sent between threads safely
   --> infra/src/media_packager/package_and_sign_processor.rs:218:5
    |
218 |     async fn process(
    |     ^^^^^ future created by async block is not `Send`
    |
    = help: the trait `Sync` is not implemented for `dyn Signer`
note: future is not `Send` as this value is used across an await
   --> infra/src/media_packager/package_and_sign_processor.rs:142:59
    |
 96 |         let signer = self.c2pa_context.signer()?;
    |             ------ has type `&dyn Signer` which is not `Send`
...
142 |             tokio::fs::create_dir_all(&signed_output_dir).await?;
    |                                                           ^^^^^ await occurs here, with `signer` maybe used later

Root Cause¶

The dyn Signer trait object does not implement Send or Sync, which is required for futures that can be sent between threads. The issue occurred because:

  1. signer was obtained early in the function
  2. An .await point occurred while signer was still in scope
  3. The compiler couldn't guarantee thread safety

Solution¶

Restructured the sign_fragmented method to ensure signer is not held across any .await points:

Before¶

async fn sign_fragmented(...) {
    // Get builder and signer EARLY
    let mut builder = c2pa::Builder::from_shared_context(&self.c2pa_context);
    let signer = self.c2pa_context.signer()?;

    for init in inits {
        // ... collect data ...

        // AWAIT while signer is in scope ❌
        tokio::fs::create_dir_all(&signed_output_dir).await?;

        builder.sign_fragmented_files(signer, ...)?;
    }
}

After¶

async fn sign_fragmented(...) {
    // Collect all jobs WITHOUT getting signer
    let mut sign_jobs = Vec::new();

    for init in inits {
        // ... collect data into sign_jobs ...
        sign_jobs.push(SignJob { init_path, fragments, output_dir });
    }

    // Do ALL async operations BEFORE getting signer
    for job in &sign_jobs {
        tokio::fs::create_dir_all(&job.output_dir).await?;
    }

    // Get builder and signer AFTER all await points
    let mut builder = c2pa::Builder::from_shared_context(&self.c2pa_context);
    let signer = self.c2pa_context.signer()?;

    // Now do all sync signing operations (no await) ✅
    for job in sign_jobs {
        builder.sign_fragmented_files(signer, &job.init_path, &job.fragments, &job.output_dir)?;
    }
}

Key Changes¶

  1. Phase 1: Collect - Gather all init segments and fragments into a Vec<SignJob>
  2. Phase 2: Prepare - Create all output directories (async, before getting signer)
  3. Phase 3: Sign - Get signer and sign all files (sync, no await points)

Benefits¶

✅ Thread Safety: Signer is never held across await points
✅ Performance: Creates directories in parallel before signing
✅ Correctness: No race conditions or Send trait violations
✅ Clarity: Clear separation of async and sync operations

Code Flow¶

1. Collect all signing jobs (no signer)
   ↓
2. Create all output directories (async, no signer)
   ↓
3. Get signer (after all awaits)
   ↓
4. Sign all files (sync, no more awaits)

Testing¶

The fix maintains the same functionality: - All init segments are still discovered - Output directories are created correctly - Signing happens in the same order - Error handling is preserved

No API changes - existing tests and curl commands work unchanged.
