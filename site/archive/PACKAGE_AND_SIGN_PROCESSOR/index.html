Package and Sign Processor¶

Overview¶

The package_and_sign processor is a combined processor that:

  1. Packages multiple resolution MP4 files into DASH/HLS format using FFmpeg and Bento4
  2. Signs the packaged fragmented MP4 content with C2PA metadata in place

This processor eliminates the need to run packaging and signing as separate steps.

Processor Kind¶

package_and_sign

Input Type¶

  • local_folder_path: A folder containing multiple MP4 files (different resolutions)

Parameters¶

Parameter Type Required Default Description
output_name string Yes - Name for the output packaged content
playlist_pattern string No **/playlist.m3u8 Glob pattern to match playlist files
init_pattern string No **/init_*.mp4 Glob pattern to match init segments
frag_pattern string No seg_*.m4s Glob pattern to match fragment segments

How It Works¶

Step 1: Packaging¶

  1. Scans the input folder for MP4 files
  2. Demuxes video and audio streams in parallel
  3. Fragments each stream with Bento4 (mp4fragment)
  4. Packages all streams into DASH/HLS format with Bento4 (mp4dash)

Step 2: C2PA Signing (In Place)¶

  1. Scans the packaged output for init segments matching init_pattern
  2. For each init segment, finds corresponding fragments matching frag_pattern
  3. Signs the init segment and fragments with C2PA metadata
  4. Writes signed content in place (no separate signed folder)

Configuration¶

The processor uses two configuration files:

  1. Pipeline Config: configs/media_packager_pipeline.toml - Controls demux, fragment, and package settings
  2. C2PA Settings: configs/default_c2pa_sdk_settings.toml - Controls C2PA signing behavior

API Endpoints¶

Asynchronous (Submit and Poll)¶

# Submit the job
curl -X POST http://localhost:8080/batches \
  -H "Content-Type: application/json" \
  -d @configs/package_and_sign_example.json

# Response: {"batch_id": "abc123..."}

# Poll for status
curl http://localhost:8080/batches/abc123...

Synchronous (Wait for Completion)¶

curl -X POST http://localhost:8080/batches/sync \
  -H "Content-Type: application/json" \
  -d @configs/package_and_sign_example.json

Request Examples¶

Minimal Request (Using Defaults)¶

{
  "inputs": [
    {
      "type": "local_folder_path",
      "path": "./test_inputs"
    }
  ],
  "processor": {
    "kind": "package_and_sign",
    "params": {
      "output_name": "my_content"
    }
  }
}

Full Request (Custom Patterns)¶

{
  "inputs": [
    {
      "type": "local_folder_path",
      "path": "./playlists/publication1"
    }
  ],
  "processor": {
    "kind": "package_and_sign",
    "params": {
      "output_name": "publication1",
      "playlist_pattern": "**/playlist.m3u8",
      "init_pattern": "**/init_*.mp4",
      "frag_pattern": "seg_*.m4s"
    }
  },
  "output": {
    "type": "local_path",
    "base": "./output/signed"
  }
}

With HTTP URL Input¶

{
  "inputs": [
    {
      "type": "http_url",
      "url": "https://example.com/media-archive.zip",
      "filename_hint": "media.zip"
    }
  ],
  "processor": {
    "kind": "package_and_sign",
    "params": {
      "output_name": "remote_content"
    }
  }
}

Input Folder Structure¶

The input folder should contain MP4 files with different resolutions:

test_inputs/
├── video_270p.mp4
├── video_360p.mp4
├── video_540p.mp4
├── video_720p.mp4
├── video_1080p.mp4
└── audio.mp4

Output Structure¶

After packaging and signing, the output will look like:

my_content_packaged_signed/
├── stream.mpd                    # DASH manifest
├── master.m3u8                   # HLS master playlist
├── video_c2pa_video_270p_frag/
│   ├── init.mp4                  # Signed init segment
│   ├── seg_00000.m4s             # Signed fragment
│   ├── seg_00001.m4s
│   └── ...
├── video_c2pa_video_720p_frag/
│   ├── init.mp4
│   └── ...
└── audio_c2pa_audio_frag/
    ├── init.mp4
    └── ...

Curl Commands¶

Package and Sign Local Files (Async)¶

curl -X POST http://localhost:8080/batches \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "type": "local_folder_path",
        "path": "./test_inputs"
      }
    ],
    "processor": {
      "kind": "package_and_sign",
      "params": {
        "output_name": "my_signed_content"
      }
    },
    "output": {
      "type": "local_path",
      "base": "./output"
    }
  }'

Package and Sign Local Files (Sync)¶

curl -X POST http://localhost:8080/batches/sync \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "type": "local_folder_path",
        "path": "./test_inputs"
      }
    ],
    "processor": {
      "kind": "package_and_sign",
      "params": {
        "output_name": "my_signed_content"
      }
    }
  }'

Using the Example Config File¶

# Async
curl -X POST http://localhost:8080/batches \
  -H "Content-Type: application/json" \
  -d @configs/package_and_sign_example.json

# Sync (blocks until complete)
curl -X POST http://localhost:8080/batches/sync \
  -H "Content-Type: application/json" \
  -d @configs/package_and_sign_example.json

Error Handling¶

Common Errors¶

Error Cause Solution
No MP4 files found in folder Input folder is empty or contains no MP4 files Ensure input folder has .mp4 files
Media packaging failed FFmpeg or Bento4 command failed Check logs for detailed error messages
C2PA signing failed No init segments found or signing error Verify patterns match packaged output
unknown processor kind Processor not registered Ensure server has loaded the processor

Environment Variables¶

Configure timeout for synchronous requests:

# Default: 120 seconds (2 minutes)
export SYNC_TIMEOUT_MS=180000

# Default: 250ms
export SYNC_POLL_MS=500

Logging¶

Control log verbosity with RUST_LOG:

# Production (INFO level, file logging)
RUST_LOG=info cargo run

# Development (DEBUG level, console only)
RUST_LOG=debug cargo run

# Full debugging (TRACE level, file + console)
RUST_LOG=trace cargo run
