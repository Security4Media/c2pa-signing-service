V1 API Deprecation and Conversions Removal - Summary¶

Date: March 4, 2026
Status: ✅ Complete

Overview¶

Successfully removed the v1 API and refactored the codebase to eliminate the need for the conversions layer by updating domain types to use chrono::DateTime directly instead of millisecond timestamps.

Changes Made¶

1. Removed V1 API¶

Deleted: - service-http/src/v1/ directory (all files) - v1_api and v2_api feature flags from service-http/Cargo.toml - Feature flag conditionals from service-http/src/main.rs

Result: Single, clean API without versioning complexity.

2. Updated Domain Types to Use DateTime¶

Modified domain/src/models/job.rs: 

// Before:
pub started_at_ms: Option<u64>,
pub finished_at_ms: Option<u64>,

// After:
pub started_at: Option<DateTime<Utc>>,
pub finished_at: Option<DateTime<Utc>>,

Benefits: - Native datetime handling - Automatic ISO 8601 serialization - Type-safe date operations - No manual timestamp conversion needed

3. Updated InMemoryJobStore¶

Modified infra/src/stores/store_inmem.rs: - Replaced now_ms() function with Utc::now() - Updated JobEntry struct to use DateTime<Utc> - Simplified timestamp handling in set_status() method

Removed: 

fn now_ms() -> u64 {
    std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .map(|d| d.as_millis() as u64)
        .unwrap_or(0)
}

4. Removed Conversions Layer¶

Deleted: - service-http/src/v2/conversions.rs - All conversion functions (snapshot_to_result, etc.)

Added From trait implementation in service-http/src/v2/dto.rs: 

impl From<BatchSnapshot> for JobResultResponse {
    fn from(snapshot: BatchSnapshot) -> Self {
        // Direct conversion without intermediate layer
        // ...
    }
}

Benefits: - Standard Rust idiom (From trait) - Less code to maintain - Clearer ownership semantics - No separate conversion module needed

5. Simplified Handlers¶

Updated service-http/src/v2/handlers.rs: 

// Before:
let result = snapshot_to_result(snapshot);

// After:
let result = JobResultResponse::from(snapshot);

Changes: - Removed conversions import - Use From trait directly - Cleaner, more idiomatic Rust

6. Updated Main Entry Point¶

Simplified service-http/src/main.rs: - Removed feature flag conditionals - Single code path for API initialization - Cleaner structure

Before: 

#[cfg(feature = "v1_api")]
mod v1;

#[cfg(feature = "v2_api")]
mod v2;

// ... complex conditional routing

After: 

mod v2;

// ... simple router creation

7. Added Dependencies¶

Workspace Cargo.toml: 

chrono = { version = "0.4", features = ["serde"] }

domain/Cargo.toml and infra/Cargo.toml: 

chrono = { workspace = true }

3. More Idiomatic Rust¶

  • Standard library patterns
  • Less boilerplate

4. Easier Maintenance¶

  • Less code to maintain
  • No conversion layer to keep in sync
  • Fewer places for bugs

5. Better Developer Experience¶

  • Direct type conversions
  • No manual timestamp math
  • Clear, readable code

Code Quality Improvements¶

Before (Conversions Layer)¶

// conversions.rs
fn millis_to_datetime(millis: u64) -> DateTime<Utc> {
    let secs = (millis / 1000) as i64;
    let nanos = ((millis % 1000) * 1_000_000) as u32;
    Utc.timestamp_opt(secs, nanos).unwrap()
}

pub fn snapshot_to_result(snapshot: BatchSnapshot) -> JobResultResponse {
    let queued_at = if let Some(started_ms) = snapshot.started_at_ms {
        millis_to_datetime(started_ms.saturating_sub(1000))
    } else {
        Utc::now()
    };
    // ... more conversion logic
}

After (Direct Conversion)¶

// dto.rs
impl From<BatchSnapshot> for JobResultResponse {
    fn from(snapshot: BatchSnapshot) -> Self {
        let queued_at = snapshot
            .started_at
            .map(|dt| dt - chrono::Duration::seconds(1))
            .unwrap_or_else(Utc::now);
        // ... clean, type-safe conversion
    }
}
