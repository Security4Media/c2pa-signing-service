API Migration Guide¶

For Users of infra¶

Before: 

use infra::{C2paProcessor, InMemoryJobStore, CompositeInputResolver};

After (recommended - more explicit): 

use infra::processors::C2paProcessor;
use infra::job_stores::InMemoryJobStore;
use infra::input_resolvers::CompositeInputResolver;

// Or use presets for common cases
use infra::presets;
let store = presets::in_memory_job_store();

Still works (backward compatible): 

use infra::{C2paProcessor, InMemoryJobStore, CompositeInputResolver};

For Users of domain¶

Before: 

use domain::processor::{Processor, ArtifactData};
use domain::models::ByteStream;

After: 

use domain::{Processor, ArtifactData};
use domain::ByteStream; // workspace-internal, marked #[doc(hidden)]

For Service HTTP¶

Before: 

// 58 lines of processor registration in processing.rs

After: 

pub use app::registry::build_default_registry as build_processor_registry;
// 5 lines total
