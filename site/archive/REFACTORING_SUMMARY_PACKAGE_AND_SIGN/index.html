Package and Sign Processor - Refactoring Summary¶

Changes Made¶

Overview¶

Refactored the package_and_sign processor to use a temporary directory for the packaging phase, then copy signed segments and manifests to the final output directory.

Key Changes¶

1. Temporary Directory for Packaging¶

  • Before: Packaged directly to final output directory
  • After: Uses tempfile crate to create a temporary directory for packaging
  • Benefit: Clean separation of packaging and signing phases, automatic cleanup

2. Sign from Source to Destination¶

  • Before: Attempted to sign in place (which is not supported by builder.sign_fragmented_files)
  • After: Signs from temporary packaging directory to final output directory
  • Method: sign_fragmented() now takes source and destination directories
  • Benefit: Properly uses the C2PA API as intended

3. Copy All Manifests and Playlists¶

  • New Feature: copy_manifests() method
  • Patterns: Copies all *.m3u8 and *.mpd files recursively
  • Directory Structure: Maintains relative paths from source to destination
  • Benefit: Ensures all playlists and manifests are included in final output

Workflow¶

Input MP4 files
    ↓
1. Package to TEMP directory
    ├── Demux (parallel)
    ├── Fragment (parallel)
    └── Package (DASH/HLS)
    ↓
2. Sign TEMP → FINAL
    ├── Find init segments in TEMP
    ├── Find fragments for each init
    ├── Sign to FINAL directory
    └── Maintain directory structure
    ↓
3. Copy manifests TEMP → FINAL
    ├── Copy all *.m3u8 files
    ├── Copy all *.mpd files
    └── Maintain directory structure
    ↓
4. Cleanup TEMP directory
    └── Automatically deleted when dropped
    ↓
Final signed output ready

File Structure¶

Temporary Directory (auto-cleaned)¶

/tmp/package_and_sign_XXXXXX/
├── stream.mpd
├── master.m3u8
├── video_AVC270_frag/
│   ├── init.mp4 (unsigned)
│   └── seg_*.m4s (unsigned)
└── audio_ST_frag/
    └── ...

Final Output Directory¶

<output_name>_packaged_signed/
├── stream.mpd (copied from temp)
├── master.m3u8 (copied from temp)
├── video_AVC270_frag/
│   ├── init.mp4 (signed)
│   └── seg_*.m4s (signed)
└── audio_ST_frag/
    └── ... (signed)

Code Changes¶

New Dependencies¶

  • infra/Cargo.toml: Added tempfile = { workspace = true }
  • package_and_sign_processor.rs: Added use tempfile::TempDir;

Modified Methods¶

sign_fragmented()¶

Before: sign_fragmented_in_place(output_dir, init_pattern, frag_pattern) After: sign_fragmented(source_dir, output_dir, init_pattern, frag_pattern)

Changes: - Takes both source and output directories - Searches for segments in source directory - Calculates relative paths to maintain structure - Signs to output directory with proper structure

copy_manifests() [NEW]¶

New method that: - Finds all *.m3u8 and *.mpd files in source - Maintains relative directory structure - Copies to destination with async I/O - Logs each copied file

process()¶

Workflow changes: 1. Create temporary directory with tempfile::Builder 2. Package to temporary directory 3. Create final output directory 4. Sign from temp to final 5. Copy manifests from temp to final 6. Drop temp_dir to trigger cleanup

Benefits¶

✅ Clean Separation: Packaging and signing are completely separate
✅ Proper C2PA API Usage: Signs from source to destination as intended
✅ Automatic Cleanup: Temp directory is automatically deleted
✅ Complete Output: All manifests and playlists are copied
✅ Directory Structure: Maintains proper hierarchy
✅ Better Logging: Clear steps with progress tracking
✅ Error Handling: Each phase can be independently error-checked

Testing¶

The processor maintains the same public API, so existing tests and curl commands remain valid:

curl -X POST http://localhost:8080/batches/sync \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [{"type": "local_folder_path", "path": "./test_inputs"}],
    "processor": {
      "kind": "package_and_sign",
      "params": {"output_name": "my_content"}
    }
  }'

Performance Impact¶

  • Minimal: Temp directory is on same filesystem (fast)
  • I/O: Additional copy operations for manifests (typically < 1 MB)
  • Cleanup: Automatic, no manual intervention needed
  • Disk: Temp space released immediately after completion

Error Scenarios¶

All errors properly handled and logged:

  1. Packaging fails: Temp dir cleaned up automatically
  2. Signing fails: Partial output in final dir, temp cleaned up
  3. Manifest copy fails: Signed segments exist, manifests may be incomplete
  4. Temp dir creation fails: Early failure, no side effects

Future Enhancements¶

Potential improvements: - Configurable temp directory location - Parallel signing of init segments - Resume capability using temp directory - Progress reporting per phase
