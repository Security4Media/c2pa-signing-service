Refactoring Complete: C2PA Code Duplication Eliminated¶

✅ Implementation Summary¶

Successfully refactored three processors to eliminate code duplication by extracting common functionality into reusable utility modules.

📦 New Files Created¶

1. infra/src/c2pa/utils.rs (293 lines)¶

Shared C2PA signing utilities: - FragmentedSignConfig - Configuration for signing patterns (init/fragment patterns) - FragmentedSignJob - Represents a single signing job - FragmentedSigner - Reusable fragmented signer with Send-safe implementation - from_sdk_toml() - Load from TOML config - collect_sign_jobs() - Find init segments and fragments - sign_all() - Sign all jobs (async, Send-safe) - sign_fragmented() - Complete workflow - ManifestCopier - Reusable manifest/playlist copier - copy_manifests() - Async version - copy_manifests_sync() - Sync version - default_patterns() - HLS/DASH patterns

2. infra/src/media_packager/utils.rs (70 lines)¶

Shared media utilities: - MediaFileScanner - Scan directories for media files - scan_mp4_files() - Scan for MP4 files - scan_media_files() - Scan for custom extensions - OutputPathBuilder - Consistent output path generation - packaged_path() - For packaging output - signed_path() - For signing output - packaged_and_signed_path() - For combined output

🔄 Refactored Files¶

1. PackageAndSignProcessor (380 → 202 lines, -47% code)¶

Before: - Duplicated C2PA context initialization - Duplicated signing logic (~100 lines) - Duplicated manifest copying (~50 lines) - Duplicated MP4 scanning

After: - Uses FragmentedSigner for all C2PA operations - Uses ManifestCopier for manifest copying - Uses MediaFileScanner for MP4 scanning - Uses OutputPathBuilder for path generation - Clean, focused on orchestration

2. MediaPackagerProcessor (104 → ~75 lines, -28% code)¶

Before: - Duplicated MP4 scanning logic - Manual output path construction

After: - Uses MediaFileScanner::scan_mp4_files() - Uses OutputPathBuilder::packaged_path() - Cleaner, more maintainable

3. C2pafMP4Processor (179 → ~95 lines, -47% code)¶

Before: - Duplicated C2PA context initialization - Duplicated signing logic (~70 lines) - Duplicated playlist copying (~30 lines) - Manual output path construction

After: - Uses FragmentedSigner for signing - Uses ManifestCopier::copy_manifests_sync() - Uses OutputPathBuilder::signed_path() - Simple and clean

🔧 Technical Highlights¶

Send-Safe Signing¶

The FragmentedSigner::sign_all() method uses a three-phase approach to avoid holding dyn Signer across await points:

  1. Phase 1: Collect - Gather all signing jobs (no signer)
  2. Phase 2: Prepare - Create directories async (no signer)
  3. Phase 3: Sign - Get signer and sign sync (no awaits)

This eliminates Send trait violations while maintaining clean async code.

Flexible Manifest Copying¶

ManifestCopier provides both async and sync versions: - Async: For use in async contexts (processors) - Sync: For compatibility with non-async code - Configurable patterns: Support any file patterns

Consistent Path Generation¶

OutputPathBuilder standardizes output naming: - <name>_packaged - For packaging-only - <name>-signed - For signing-only
- <name>_packaged_signed - For combined operations
