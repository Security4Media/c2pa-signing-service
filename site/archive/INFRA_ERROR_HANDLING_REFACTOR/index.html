Infra Crate Error Handling Refactor¶

Overview¶

Refactored the infra crate to use thiserror for structured, general error handling that provides clear error categories without exposing implementation details.

Date: March 9, 2026
Scope: crates/infra/src/error/
Architecture: Hexagonal (Ports & Adapters)

Goals Achieved¶

  1. ✅ Type-safe error handling using thiserror with #[from] attribute
  2. ✅ General error categories - no overly specific error variants
  3. ✅ Modular organization - separate files per concern for clarity
  4. ✅ Automatic error conversion - leveraging #[from] for common cases
  5. ✅ Clean error messages - simple, non-leaking error descriptions
  6. ✅ Backward compatibility - automatic conversion to anyhow::Error

Architecture¶

Module Structure¶

crates/infra/src/error/
├── mod.rs            # Top-level InfraError + exports
├── processor.rs      # Processor & C2PA errors
├── resolver.rs       # Input resolution errors
├── store.rs          # Job storage errors
└── sink.rs           # Output sink errors

Key Design Principles¶

1. General Error Categories¶

Errors are categorized by concern, not by specific failure modes:

// ✅ General
#[error("I/O operation failed")]
Io { #[from] source: std::io::Error }

// ❌ Too specific (avoided)
#[error("Failed to write file: {path}")]
FileWrite { path: String, source: std::io::Error }

2. Automatic Conversion with #[from]¶

Use #[from] attribute for automatic error conversion:

#[derive(Error, Debug)]
pub enum ProcessorError {
    #[error("I/O operation failed")]
    Io {
        #[from]
        source: std::io::Error,
    },
}

// Usage - automatic conversion!
fn process_file() -> Result<(), ProcessorError> {
    let file = std::fs::File::open("test.txt")?;  // io::Error auto-converts
    Ok(())
}

3. Modular Organization¶

Each error module is self-contained: - processor.rs - Processor and C2PA errors - resolver.rs - Input resolution errors - store.rs - Job storage errors - sink.rs - Output sink errors - mod.rs - Top-level type and exports

4. No Redundant Constructors¶

Errors use direct enum construction - no helper methods needed:

// ✅ Direct construction
return Err(ProcessorError::UnsupportedFormat);
return Err(ResolverError::PathNotFound);

// ❌ Redundant helpers (removed)
// ProcessorError::unsupported_format()
// ResolverError::path_not_found()

Error Categories¶

Processor Errors¶

Purpose: Errors during processor execution (C2PA, media packaging)

ProcessorError::C2paSigning           // C2PA signing failed

Purpose: C2PA SDK-specific errors

Resolver Errors¶

ResolverError::InvalidUrl          // Invalid URL
ResolverError::UnsupportedScheme   // Unsupported URL scheme

ResolverError::StreamConversion    // Stream conversion failed
ResolverError::InvalidContentType  // Invalid content type

Store Errors¶

Purpose: Job storage operation failures

StoreError::JobNotFound         // Job not found

Purpose: Output sink operation failures

Usage Examples¶

Basic Error Handling¶

use infra::ProcessorError;

fn process_media() -> Result<(), ProcessorError> {
    // I/O errors auto-convert via #[from]
    let data = std::fs::read("input.mp4")?;

    if data.is_empty() {
        return Err(ProcessorError::UnsupportedFormat);
    }

    Ok(())
}

Error Conversion Chain¶

// c2pa::Error -> C2paError -> ProcessorError -> InfraError
fn sign_media() -> Result<(), InfraError> {
    let signer = get_signer()?;  // c2pa::Error
    signer.sign()?;               // Automatically converted through chain
    Ok(())
}

Pattern Matching¶

match result {
    Ok(artifact) => process_artifact(artifact),
    Err(ProcessorError::C2paSigning { source }) => {
        error!("C2PA signing failed: {}", source);
    }
    Err(ProcessorError::InvalidParams { message, .. }) => {
        warn!("Invalid params: {}", message);
    }
    Err(e) => {
        error!("Processing failed: {}", e);
    }
}

Conversion to anyhow¶

// Automatic conversion (thiserror implements std::error::Error)
fn legacy_function() -> anyhow::Result<()> {
    process_media()?;  // ProcessorError -> anyhow::Error automatically
    Ok(())
}

  • ❌ Redundant constructor methods

After Refactor¶

  • ✅ Type-safe, structured errors
  • ✅ Modular organization (5 focused files)
  • ✅ Automatic error conversion via #[from]
  • ✅ General error categories
  • ✅ Clean, simple error messages
  • ✅ Direct enum construction
  • ✅ Backward compatible with anyhow

Integration with Other Layers¶

Domain Layer¶

// Domain uses anyhow::Result
pub trait Processor {
    async fn process(&self, input: I, ctx: ProcessCtx) -> anyhow::Result<Artifact>;
}

Infra Implementation¶

#[async_trait]
impl Processor for C2paFileProcessor {
    async fn process(&self, input: I, ctx: ProcessCtx) -> anyhow::Result<Artifact> {
        self.sign_monolithic(input).await?;  // ProcessorError -> anyhow automatically
        Ok(artifact)
    }
}

App Layer¶

// App converts infra errors to JobError
let artifact = processor
    .process(input, ctx)
    .await
    .map_err(JobError::processor_execution)?;

Service Layer¶

// Service converts to ApiError
let batch_id = state.service
    .submit(spec)
    .await?;  // AppError -> ApiError automatically

Best Practices¶

1. Use #[from] for Common Conversions¶

#[derive(Error, Debug)]
pub enum ProcessorError {
    #[error("I/O operation failed")]
    Io {
        #[from]  // Automatic conversion from io::Error
        source: std::io::Error,
    },
}

2. Keep Error Messages General¶

// ✅ Good - general
#[error("HTTP request failed")]
HttpRequest { source: reqwest::Error }

// ❌ Too specific - avoid
#[error("HTTP request to {url} failed with status {status}")]
HttpRequestFailed { url: String, status: u16, source: reqwest::Error }

3. Use Direct Construction¶

// ✅ Good - direct
return Err(ResolverError::PathNotFound);

// ❌ Redundant - avoid
return Err(ResolverError::path_not_found());

4. Preserve Error Context¶

#[derive(Error, Debug)]
pub enum ProcessorError {
    #[error("Invalid processor parameters: {message}")]
    InvalidParams {
        message: String,
        #[source]  // Preserves underlying error
        source: Option<serde_json::Error>,
    },
}

Migration Notes¶

For New Code¶

  • Use ProcessorError, ResolverError, etc. directly
  • Leverage ? operator for automatic conversion
  • Use #[from] for common error types

For Existing Code¶

  • Errors automatically convert to anyhow::Error
  • No breaking changes for domain/app layers
  • Gradual migration possible

Testing Recommendations¶

#[test]
fn test_error_conversion() {
    let io_err = std::io::Error::new(std::io::ErrorKind::NotFound, "test");
    let proc_err: ProcessorError = io_err.into();
    assert!(matches!(proc_err, ProcessorError::Io { .. }));
}

#[test]
fn test_error_chain() {
    let c2pa_err = C2paError::Sdk { source: c2pa::Error::BadParam("test".into()) };
    let proc_err: ProcessorError = c2pa_err.into();
    assert!(matches!(proc_err, ProcessorError::C2paSigning { .. }));
}

#[test]
fn test_anyhow_conversion() {
    let proc_err = ProcessorError::UnsupportedFormat;
    let anyhow_err: anyhow::Error = proc_err.into();
    assert_eq!(anyhow_err.to_string(), "Unsupported media format");
}

Extension Example¶

#[derive(Error, Debug)]
pub enum ProcessorError {

impl ProcessorError {
    /// Get error code for metrics/logging
    pub fn code(&self) -> &'static str {
        match self {
            ProcessorError::Io { .. } => "PROC_IO",
            ProcessorError::C2paSigning { .. } => "PROC_C2PA",
            // ...
        }
    }

    }
}
