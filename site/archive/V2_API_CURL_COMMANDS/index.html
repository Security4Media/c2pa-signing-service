API - Example cURL Commands¶

Health Check¶

curl -X GET http://localhost:8080/v2/health

C2PA Video Signing¶

Async (returns job_id immediately)¶

curl -X POST http://localhost:8080/v2/c2pa/video \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "type": "local_file_path",
        "path": "/app/input/video1.mp4"
      },
      {
        "type": "local_file_path",
        "path": "/app/input/video2.mp4"
      }
    ],
    "params": {
      "media_hint": "video/mp4",
      "output_name": "signed_{original}"
    },
    "output": {
      "type": "local_path",
      "base": "/app/output"
    },
    "options": {
      "max_files_parallel": 2
    }
  }'

Sync (blocks until completion)¶

curl -X POST http://localhost:8080/v2/sync/c2pa/video \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "type": "local_file_path",
        "path": "/app/input/video1.mp4"
      }
    ],
    "params": {
      "media_hint": "video/mp4"
    },
    "output": {
      "type": "local_path",
      "base": "/app/output"
    }
  }'

C2PA Fragmented (HLS) Signing¶

Async¶

curl -X POST http://localhost:8080/v2/c2pa/fragmented \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "type": "hls_publication",
        "path": "/app/playlists/publication1"
      }
    ],
    "params": {
      "output_name": "publication1_signed",
      "playlist_pattern": "**/playlist.m3u8",
      "init_pattern": "**/init_*.mp4",
      "frag_pattern": "seg_*.m4s"
    },
    "output": {
      "type": "local_path",
      "base": "/app/output"
    }
  }'

Sync¶

curl -X POST http://localhost:8080/v2/sync/c2pa/fragmented \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": [
      {
        "type": "local_folder_path",
        "path": "/app/output/packaged_output"
      }
    ],
    "params": {
      "output_name": "signed_output"
    },
    "output": {
      "type": "local_path",
      "base": "/app/output"
    }
  }'

Job Status Query¶

Get job status (for async requests)¶

# After submitting an async request, you'll get a response like:
# {"job_id":"uuid-here","status":"queued","status_url":"/v2/jobs/uuid-here"}

# Query the job status:
curl -X GET http://localhost:8080/v2/jobs/uuid-here

Configuration¶

Environment variables:

  • SYNC_TIMEOUT_MS - Timeout for sync endpoints (default: 120000 ms / 2 minutes)
  • SYNC_POLL_MS - Polling interval for sync endpoints (default: 250 ms)
  • RUST_LOG - Log level (e.g., info, debug, trace)

Building & Running¶

# Build the service
cargo build --package service-http

# Run the service
cargo run --package service-http

Input Types Reference¶

All endpoints support these input types:

  • local_file_path - File on local filesystem
  • local_folder_path - Folder on local filesystem
  • hls_publication - HLS publication structure
  • http_url - HTTP/HTTPS URL
  • s3 - AWS S3 object
  • gcs - Google Cloud Storage object
  • drive - Google Drive file
  • in_request - Multipart upload (not yet implemented for v2)

Notes¶

  • All endpoints accept arrays of inputs for batch processing
  • Sync endpoints will timeout after SYNC_TIMEOUT_MS (default 2 minutes)
  • Partial results are treated as failures
  • Job results include per-input status and error messages
