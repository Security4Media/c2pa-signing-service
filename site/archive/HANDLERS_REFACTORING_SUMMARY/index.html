Handlers Refactoring Summary¶

Date: March 4, 2026

Overview¶

Successfully refactored the service-http/src/v2/handlers.rs monolithic file into a well-organized module structure with the From trait pattern for type conversions.

Changes Made¶

1. Created New Module Structure¶

Split the single handlers.rs file into logical submodules:

crates/service-http/src/v2/
├── conversions.rs              # NEW: From/TryFrom implementations
├── handlers/                   # NEW: Module directory
│   ├── mod.rs                  # Re-exports
│   ├── common.rs               # Shared job submission logic
│   ├── c2pa_video.rs           # C2PA video signing handlers
│   ├── c2pa_fragmented.rs      # C2PA fragmented/HLS handlers

│   └── status.rs               # Job status & health check
└── handlers.rs                 # MODIFIED: Now just re-exports

2. Introduced From Pattern for Type Conversions¶

Before (Procedural)¶

fn to_batch_spec(
    inputs: Vec<domain::InputRef>,
    processor_kind: &str,
    params: serde_json::Value,
    output: Option<OutputSpec>,
    options: ProcessingOptions,
) -> BatchSpec {
    BatchSpec {
        inputs,
        processor: ProcessorSpec {
            kind: processor_kind.to_string(),
            params,
        },
        output: output.map(|o| match o {
            OutputSpec::LocalPath { base } => DomainOutputSpec::LocalPath { base },
        }),
        options: BatchOptions {
            max_files_parallel: options.max_files_parallel,
        },
    }
}

After (Idiomatic Rust)¶

// conversions.rs

// Direct conversions
impl From<ProcessingOptions> for BatchOptions { ... }
impl From<OutputSpec> for DomainOutputSpec { ... }

// Builder pattern for complex conversions
pub struct BatchSpecBuilder {
    pub processor_kind: String,
    pub params: serde_json::Value,
}

impl TryFrom<C2paVideoRequest> for BatchSpecBuilder { ... }
impl TryFrom<C2paFragmentedRequest> for BatchSpecBuilder { ... }
impl TryFrom<PackageRequest> for BatchSpecBuilder { ... }
impl TryFrom<PackageAndSignRequest> for BatchSpecBuilder { ... }

3. Extracted Common Logic¶

handlers/common.rs¶

  • submit_job_async() - Submit job and return immediately
  • submit_job_sync() - Submit job and poll until completion

4. Organized by Feature¶

Each handler module focuses on a single endpoint group:

  • c2pa_video.rs - Video signing (sync/async)

  • package_and_sign.rs - Combined operation (sync/async)
  • status.rs - Job status queries & health check

Pattern Examples¶

Using From Pattern in Handlers¶

Before: 

pub async fn c2pa_video_async(...) -> ... {
    let params = serde_json::to_value(&req.params)
        .map_err(|e| ApiError::internal_error(e.to_string()))?;
    let spec = to_batch_spec(req.inputs, "c2pa_sign", params, req.output, req.options);
    submit_job_async(&state, spec).await
}

After: 

pub async fn c2pa_video_async(...) -> ... {
    let builder: BatchSpecBuilder = req.clone().try_into()
        .map_err(|e: serde_json::Error| ApiError::internal_error(e.to_string()))?;
    let spec = builder.build(req.inputs, req.output, req.options);
    submit_job_async(&state, spec).await
}

Builder Pattern¶

// In conversions.rs
pub struct BatchSpecBuilder {
    pub processor_kind: String,
    pub params: serde_json::Value,
}

impl BatchSpecBuilder {
    pub fn build(
        self,
        inputs: Vec<domain::InputRef>,
        output: Option<OutputSpec>,
        options: ProcessingOptions,
    ) -> BatchSpec {
        BatchSpec {
            inputs,
            processor: ProcessorSpec {
                kind: self.processor_kind,
                params: self.params,
            },
            output: output.map(|o| o.into()),
            options: options.into(),
        }
    }
}

Migration Notes¶

For Developers¶

No action required - this is an internal refactoring. All public APIs remain unchanged.

For Tests¶

If you have tests that directly import from handlers.rs, they will continue to work due to re-exports in handlers/mod.rs.

For New Handlers¶

When adding a new handler:

  1. Create new file in handlers/ (e.g., image.rs)
  2. Implement handler functions
  3. Add mod image; to handlers/mod.rs
  4. Re-export handlers: pub use image::{...};
  5. Add conversion in conversions.rs if needed

Example: 

// handlers/image.rs
pub async fn c2pa_image_async(...) -> ... {
    // Implementation
}

// handlers/mod.rs
mod image;
pub use image::{c2pa_image_async, c2pa_image_sync};
